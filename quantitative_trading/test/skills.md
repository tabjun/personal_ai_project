# 📜 Analysis Skills & Guidelines (Test Environment Only)

본 테스트 환경에서 분석을 수행할 때 반드시 준수해야 하는 원칙과 기술적 가이드라인입니다.

## 0. 현재 분석의 목적 (Current Objective)
**본 디렉토리의 현재 목적은 '퀀트 트레이딩' 시스템 구축이 아니라, 종목에 대한 '시계열 가격 예측 알고리즘 분석'입니다.** 
- 다양한 알고리즘을 테스트하여 가격 예측의 정확도와 강건성을 평가합니다.
- 이 분석 결과를 토대로 추후 퀀트 트레이딩을 위한 기준(Threshold, Strategy)을 수립하는 전단계입니다.

## 1. 데이터 관리 및 문헌 추적 원칙 (Data & Literature Principles)
- **데이터 분할**: 3년(15분봉) 데이터를 활용하며, 반드시 **2년 학습(Train) / 1년 예측(Test)** 구조를 유지합니다. (보고서의 '최근 1년'은 이 예측 기간을 의미함)
- **학술 트렌드 반영**: 새로운 알고리즘 추가 요청 시 반드시 **Antigravity CLI (agy)**의 **Arxiv MCP** 및 로컬 **Hugging Face Skills**(`.agents/skills/`)를 활용하여 최신 문헌을 추적하고 그 내용을 반영합니다.

## 2. 코드 관리 및 커밋 전략 (Dual-File Tracking)
Jupyter Notebook(`.ipynb`)의 변경 이력 추적(Diff) 어려움을 해결하기 위해 다음 전략을 **강제**합니다.
- **실시간 동시 수정**: 모든 `.ipynb` 수정 시 동일한 내용의 `.py` 파일(ipynb-py-convert 기반 `# %%` 마커 포함)을 즉시 업데이트합니다.
- **커밋용 헤더 명시**: 모든 `.py` 파일 최상단에 `# [FOR COMMIT TRACKING ONLY - DO NOT EXECUTE]` 라는 주석을 명시하여 실행용이 아님을 분명히 합니다.
- **수행 원칙**: 실제 분석 수행은 `.ipynb`에서 진행하며, `.py` 파일은 오직 **커밋 및 변경점 추적용**으로만 관리합니다.

## 3. 실험 및 검증 프로세스 (Advanced Metrics)
기존의 MSE, MAE와 같은 점 단위 오차를 넘어, 학술계에서 권장하는 다음 지표들을 추가하여 모델의 '예측 실효성'을 평가합니다.
- **Directional Accuracy (DA / Hit Ratio)**: 
  - **설명**: 가격의 상승/하락 방향을 맞춘 비율. 
  - **해석**: MSE가 낮아도 방향이 틀리면 트레이딩 가치가 0입니다. 2025년 연구 기준 54~60%의 DA는 매우 우수한 신호로 간주됩니다.
- **MASE (Mean Absolute Scaled Error)**:
  - **설명**: 단순 Persistence 모델(전날 가격이 오늘도 유지된다는 가정) 대비 오차 비율.
  - **해석**: 1보다 작으면 단순 예측보다 우수함을 의미하며, 데이터의 스케일에 무관하게 모델의 '예측 능력' 자체를 평가하는 2026년 표준 지표입니다.
- **DTW-based Similarity (Shape Matching)**:
  - **설명**: 점 단위 비교가 아닌, 예측 곡선과 실제 곡선의 '형태적 유사도'를 평가 (Dynamic Time Warping).
  - **해석**: 가격의 흐름(Trend) 자체를 얼마나 잘 복제했는지 파악하여, 지연 현상(Lagging)이 얼마나 해소되었는지 시각적/수치적으로 입증합니다.

## 4. 모델링 가이드라인 (Modeling Protocols)
- **불규칙성 모델링**: 데이터에 노이즈가 많거나 간격이 일정하지 않을 경우 **mTAND** 계열을 우선 검토합니다.
- **장기 의존성**: 수천 스텝 이상의 과거 데이터를 참조해야 할 경우 **Mamba** 또는 **TCN**을 활용합니다.
- **손실 함수**: 급격한 펌핑/덤핑과 같은 이상치에 대응하기 위해 `MSE` 단독 사용보다는 `HuberLoss`나 `Log-Cosh`를 혼합 사용합니다.
- **Shallow but Wide 원칙**: 오버피팅 억제를 위해 레이어를 1~2개 층으로 제한하되 가로 노드(64~128)를 넓히는 기조를 유지합니다.

## 5. 실험 및 검증 프로세스 (Verification Guidelines)
- 새로운 알고리즘을 추가할 때는 반드시 `research_materials/`에 최신 논문 근거를 정리합니다.
- 논문 탐색 시 **Arxiv MCP**와 **Hugging Face Skills**를 적극 활용하여 최신 SOTA(State-of-the-Art) 모델과의 성능을 비교합니다.
- 모든 그래프 시각화 시 X축(Time)과 Y축(Price/Unit)을 명시하고, 저장 경로를 `test/images/`로 설정합니다.
- 성능 지표(RMSE, MAE)는 반드시 전처리 이전의 원본 스케일로 역변환(Inverse Transform)하여 기록합니다.

## 6. 결과 보고 및 아카이빙 (Reporting Guidelines)
에이전트가 분석 결과를 바탕으로 `results/`에 리포트를 작성할 때는 루트의 `.antigravityrules` 전사 규격에 맞춰 다음 템플릿과 원칙을 엄격히 준수해야 합니다.

### [필수 포함 항목 템플릿]
1. **0. 실행 및 분석 환경**: 분석 환경(OS, GPU 등), 모델 파라미터 세팅, 데이터 수집 파이프라인(DuckDB 등).
2. **1. 기초 통계량 분석 (Descriptive Statistics)**: 학습/테스트 데이터의 `describe()` 결과 및 시계열적 특징 분석.
3. **2. 알고리즘 성능 지표**: Mamba, mTAND 등 추가된 모든 최신 알고리즘의 예측 성능(RMSE, MAE 등). 성능은 전처리 이전 스케일(역변환) 기준으로 기재합니다. (DA, MASE 지표 필수 산출)
4. **3. 시각화 결과 및 상세 해석 (축 및 범례 포함)**:
   - 그래프가 포함될 경우 반드시 **X축(예: Time in 15-min intervals)**과 **Y축(예: Price in KRW)**의 설정과 범례를 명시하고 텍스트로 축-범례 정밀 해석을 덧붙입니다.
   - 각 모델의 추세 추종성, 이상치 강건성, 지연 현상(Lagging) 해소 여부에 대해 정성적/기술적 해석을 상세히 기재합니다.
5. **4. 종합 결론 및 전략 제언**: 퀀트 트레이딩(최하방 방어) 관점에서 어떤 모델이 실전에 적합한지 결론을 내립니다.
6. **5. 주요 도메인 용어 해설 (Glossary)**: 리포트 내에 사용된 전문 용어(예: 엣지(Edge), 라깅(Lagging), MDD 등)에 대한 명확한 해설을 첨부합니다.

- 분석 완료 후 위 템플릿을 적용한 핵심 인사이트는 `test/results/analysis_report_YYYYMMDD_HHMM.md` 형식의 고유 파일로 작성합니다. 
- 기존에 작성된 `advanced_analysis_report.md`는 참고용 샘플 템플릿이며, 실제 분석 결과는 항상 새로운 타임스탬프 파일로 생성됩니다.
- 모델 파일은 `models/` 폴더 내에 버전별로 관리합니다.
