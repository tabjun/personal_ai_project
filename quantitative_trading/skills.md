# 에이전트 기술 가이드

이 문서는 이 프로젝트에서 에이전트가 참고할 기술 철학, 분석 기준, 권장 워크플로우를 정리한다.
강제 규칙은 `AGENTS.md`가 우선한다.

## 1. 핵심 기술 축

### 다변량 시계열 분석

- 가격, 거래량, 변동성만 보지 않는다.
- 뉴스, 리포트, SNS, 매크로, 유동성, 오더북, 온체인, 크로스마켓 요인을 함께 본다.
- 기본 도구: `pandas`, `numpy`, `statsmodels`, `scikit-learn`, `duckdb`

### 패턴 유사도 분석

- 단순 유클리디안 거리보다 형태 유사도를 우선한다.
- DTW 또는 유사한 시계열 정렬 기반 접근을 우선 검토한다.
- historical analog는 차트 모양만이 아니라 독립변수와 컨텍스트까지 포함해 비교한다.

### 문맥 감성 분석

- 뉴스, 증권사 리포트, SNS 텍스트를 15분 또는 분석 단위 버킷으로 정렬한다.
- 감성 점수, 이벤트 수, shock, 리스크/매크로/규제/유동성 토픽을 독립변수 후보로 쓴다.

### 방어 우선 백테스트

- 목표는 수익률 최대화보다 MDD 제어와 하방 방어다.
- 매수 신호보다 손실 제한과 진입 차단 로직의 품질을 더 엄격하게 본다.

## 2. 시계열 전처리 원칙

- 원시 가격만 그대로 밀어넣지 않는다.
- 정상성 검정과 롤링 드리프트 점검을 먼저 한다.
- log return, diff, rolling z-score, EMA 계열 변형을 비교한다.
- Lag-1 shift처럼 직전 값을 복사하는 편법 경로를 경계한다.
- 평가 수치는 KRW 원본 스케일로 역복원해 본다.

## 3. 보고서 기준

보고서와 결과 해석은 아래 원칙을 따른다.

- 실행 및 분석 환경, 기초 통계량, 방법론 정의, 성능 지표, 시각화 해석을 모두 포함한다.
- 결과만 나열하지 말고, 사용한 방법론의 개념과 왜 썼는지, 어떻게 해석하는지 먼저 설명한다.
- 수치 해석은 KRW 원본 스케일과 baseline 비교를 함께 본다.
- 최종적인 세부 형식과 예시는 `test/README.md`의 보고서 기준을 따른다.

## 4. 권장 워크플로우

1. `AGENTS.md`, `process.md`, `history.md`, `conversation_l2_cache.md`를 읽는다.
2. 기존 코드와 문서를 먼저 검색한다.
3. 연구 작업이면 `test/models/*.ipynb`를 먼저 수정하고 `.py` 미러를 맞춘다.
4. 운영/실사용 기능이면 루트 패키지와 `pipelines/`를 수정한다.
5. 경량 검증만 수행한다.
6. 결과를 `history.md`, `process.md`, 필요 시 `conversation_l2_cache.md`에 기록한다.

## 5. 문서 역할 분리

- `AGENTS.md`: 강제 규칙, 실행 경계, 세션 시작 규칙
- `skills.md`: 기술 철학, 분석 원칙, 보고서 기준
- `README.md`: 사람용 프로젝트 개요, 폴더 구조, 실행 방법
- `conversation_l2_cache.md`: 최근 사용자 의도와 제약의 압축 캐시
